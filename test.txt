The layout code is no longer the main problem.

Your preview is still breaking because the list item text is still inside a block wrapper from TinyMCE, usually <p> or sometimes <div>, so even with flex the text starts on the next line.

That means this is not fixed by row/body/marker alone.
You need to unwrap the content before appending it.

Do this exact fix

Replace your normalizeContentNodes() with this

private normalizeContentNodes(nodes: ChildNode[]): ChildNode[] {
  const normalized: ChildNode[] = [];

  for (const node of nodes) {
    if (node.nodeType === Node.TEXT_NODE) {
      const text = node.textContent?.replace(/\s+/g, ' ').trim() ?? '';
      if (text) {
        normalized.push(document.createTextNode(text));
      }
      continue;
    }

    if (!(node instanceof HTMLElement)) {
      normalized.push(node);
      continue;
    }

    const tag = node.tagName.toLowerCase();

    // unwrap block wrappers like <p> or <div> inside list item content
    if (tag === 'p' || tag === 'div') {
      const text = node.textContent?.replace(/\s+/g, ' ').trim() ?? '';
      if (text) {
        normalized.push(document.createTextNode(text));
      }
      continue;
    }

    // inline elements can stay
    node.style.margin = '0';
    normalized.push(node);
  }

  return normalized;
}

⸻

Important change here

Notice I changed:

const body = document.createElement('div');

to:

const body = document.createElement('span');

That matters.

A div is block-level.
A span keeps the first line content inline with the number more reliably.

Since the parent row is flex, the span can still behave as a flex item.

Also clean applyElementStyles()

Right now you are styling paragraphs globally:

This is reintroducing block spacing into list items.

Replace that paragraph block with this:

const paragraphs = root.querySelectorAll('p');
for (const paragraph of paragraphs) {
  const element = paragraph as HTMLElement;

  if (element.closest('li')) {
    element.style.margin = '0';
    element.style.display = 'inline';
  } else {
    element.style.margin = '0 0 0.75rem';
  }

  element.style.color = '#4c5b6b';
}


⸻

Why it is still happening

Because your list item likely looks like this before normalization:

<li>
  <p>fsdfdfds</p>
</li>

So even if you append it into the right-side container, the <p> still behaves like a block and starts on a new line.

You want it to become effectively:

<li>
  1. fsdfdfds
</li>

not:

<li>
  1.
  <p>fsdfdfds</p>
</li>


⸻

Final 3 fixes you need
	1.	normalizeContentNodes() must unwrap <p> and <div>
	2.	body should be a span, not div
	3.	applyElementStyles() must not apply block paragraph spacing inside li
