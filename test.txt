Here’s a concise but detailed prompt you can give to your LLM:

⸻

Prompt:

I need to implement conditional clause rendering based on payload values and Drools evaluation results.

Context

Each clause contains multiple variables (e.g., 5 properties). For every clause:

* Determine whether each variable is:
    * Present
    * Missing
    * Null
* Determine whether the Drools condition evaluation passes or fails for the available variables.

Based on these conditions, classify the clause into one of the following states and decide what should be displayed in the template preview/export.

Rules

1. Fully Satisfied
    * All expected variables are present.
    * No variable is null.
    * Drools evaluation passes.
    * Display: Clause content only (do not show clause name).
2. Fully Dissatisfied
    * All expected variables are present.
    * Drools evaluation fails (one or more conditions do not match).
    * Display: Nothing (hide both clause name and content).
3. Partially Dissatisfied
    * One or more variables are missing or null.
    * Among the available variables, Drools evaluation does not fully pass.
    * Display: Nothing (hide both clause name and content).
4. Partially Satisfied
    * One or more variables are missing or null.
    * All available (non-null) variables satisfy the Drools conditions.
    * Display: Clause name + Clause content.
5. Can’t Say
    * All expected variables are null (or no usable values are available).
    * Drools evaluation cannot be meaningfully performed.
    * Display: Clause name + Clause content.

Implementation Requirements

* Detect expected variables from the clause definition.
* Compare them with the incoming payload.
* Track:
    * Total expected variables
    * Present variables
    * Missing variables
    * Null variables
    * Drools evaluation result
* Implement a clean decision method (or enum-based state machine) that returns one of:
    * FULLY_SATISFIED
    * FULLY_DISSATISFIED
    * PARTIALLY_SATISFIED
    * PARTIALLY_DISSATISFIED
    * CANT_SAY
* The rendering layer should simply use this returned status to decide whether to display:
    * Content only
    * Clause name + content
    * Nothing

Write clean, maintainable, and extensible code with clear separation between evaluation logic and rendering logic.
