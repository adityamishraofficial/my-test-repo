This is mainly due to PrimeNG v18 calendar icon rendering/style change. Your current CSS only has:

.p-calendar {
  display: flex;
}

But after v18, the input + icon layout needs explicit styling, and you should use iconDisplay=“button” if you want the green calendar button on the right like before.

Use this.

calendar.component.html

<label
  [attr.for]="id"
  *ngIf="label"
>
  {{ label | translate }}
</label>
<p-calendar
  #calendar
  [(ngModel)]="value"
  [disabled]="disabled"
  [selectionMode]="selectionMode"
  [attr.id]="id"
  [showIcon]="true"
  iconDisplay="button"
  [dateFormat]="dateFormat"
  [monthNavigator]="true"
  [yearNavigator]="true"
  [yearRange]="yearRange"
  [showTime]="showTime"
  [showSeconds]="showSeconds"
  [hourFormat]="hourFormat"
  styleClass="ng-wings-calendar"
  (onBlur)="touch()"
></p-calendar>

calendar.component.scss

@import './../../../../styles/variables/all.scss';
$theme: $color-green;
:host {
  display: block;
  width: 100%;
}
:host ::ng-deep {
  .ng-wings-calendar {
    width: 100%;
    display: flex;
  }
  .ng-wings-calendar .p-inputtext {
    flex: 1;
    width: 100%;
    height: 38px;
    border-radius: 4px 0 0 4px;
  }
  .ng-wings-calendar .p-datepicker-trigger {
    width: 38px;
    min-width: 38px;
    height: 38px;
    background: $theme;
    border-color: $theme;
    color: #fff;
    border-radius: 0 4px 4px 0;
  }
  .ng-wings-calendar .p-datepicker-trigger:hover {
    background: $theme;
    border-color: $theme;
  }
  .ng-wings-calendar .p-button-icon {
    color: #fff;
  }
}

Also confirm these are imported in your global styles.scss:

@import "primeicons/primeicons.css";
@import "primeng/resources/primeng.min.css";

For PrimeNG v18 theme, make sure your theme import is also present, depending on your setup. Without the theme, PrimeNG components will look broken after migration.
