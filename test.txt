Now functionality is correct, only the alignment/styling is broken.

In PrimeNG v18, utility classes like:

flex
items-center
gap-4
ml-2

work only if you installed PrimeFlex or another utility CSS library.

Right now those classes are not applied, so labels are misaligned.

Add proper CSS manually instead.

Use this exact code:

<div class="radio-group">
  <div class="radio-item">
    <p-radioButton
      name="contractTemplateAvailable"
      [value]="true"
      formControlName="contractTemplateAvailable"
      inputId="contractTemplateAvailableYes">
    </p-radioButton>
    <label for="contractTemplateAvailableYes">Yes</label>
  </div>
  <div class="radio-item">
    <p-radioButton
      name="contractTemplateAvailable"
      [value]="false"
      formControlName="contractTemplateAvailable"
      inputId="contractTemplateAvailableNo">
    </p-radioButton>
    <label for="contractTemplateAvailableNo">No</label>
  </div>
</div>

Component SCSS/CSS:

.radio-group {
  display: flex;
  align-items: center;
  gap: 20px;
}
.radio-item {
  display: flex;
  align-items: center;
  gap: 8px;
}

OR if you want PrimeFlex utilities back like before migration:

Install:

npm install primeflex

Then add in styles.scss:

@import "primeflex/primeflex.css";

Then your old utility classes (flex, gap-4, ml-2) will work again properly.
