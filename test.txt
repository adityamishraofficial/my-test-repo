The failure is not because of PrimeNG.

Your ng update is stopping because Angular CLI checks the installed dependency tree, and it cannot find @angular-builders/jest@^18.0.0 in node_modules, even though it is present in package.json. Angular’s update command does this dependency scan before running migrations. The Angular CLI also supports upgrading by targeting the major version directly with ng update @angular/cli@^18 @angular/core@^18.  ￼

Also, Angular 18 expects compatible versions of Node, TypeScript, and RxJS. Angular’s official compatibility table for v18 allows Node ^18.19.1 || ^20.11.1 || ^22.0.0, TypeScript >=5.4.0 <5.6.0, and RxJS ^6.5.3 || ^7.4.0. Your screenshots show typescript: ^5.4.2, rxjs: ^7.8.1, and zone.js: ^0.14.10, which are in the right range for Angular 18.  ￼

PrimeNG does not need to be changed right now. PrimeTek stated they released PrimeNG 17.18.0 with peer dependency fixes so PrimeNG 17 can work with both Angular 17 and Angular 18.  ￼

Use this approach.

1) Keep PrimeNG as-is

Do not change:

"primeng": "^17.18.0"

2) Fix the real blocker first

Because @angular-builders/jest is “missing”, install dependencies cleanly before running ng update.

Run this from the project root:

rm -rf node_modules package-lock.json
npm cache verify
npm install

If you are on Windows PowerShell:

rmdir /s /q node_modules
del package-lock.json
npm cache verify
npm install

If npm install fails specifically on Jest packages, install them explicitly first:

npm install -D @angular-builders/jest@^18.0.0 jest-preset-angular@^14.2.2 jest@^29.7.0 jest-environment-jsdom@^29.7.0 @types/jest@^29.5.12

@angular-builders/jest exists and has Angular-aligned major releases on npm, and jest-preset-angular is the standard preset for Angular + Jest setups.  ￼

3) Then run the Angular update

After npm install succeeds:

npx ng update @angular/core@^18 @angular/cli@^18

Then update Angular Material/CDK if you use them:

npx ng update @angular/cdk@^18

5) Why this is happening in your case

Most likely one of these is true:
	•	you edited package.json manually but did not run npm install
	•	node_modules and package-lock.json are stale
	•	npm install previously failed, so @angular-builders/jest never got installed
	•	there is a dependency resolution issue caused by another package, and Angular only reports the first missing required package

6) Best order to migrate safely

Run these in order:

rm -rf node_modules package-lock.json
npm install
npx ng version
npx ng update @angular/core@^18 @angular/cli@^18
npm install
npx ng build
npx ng test
