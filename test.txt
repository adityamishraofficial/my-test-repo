In PrimeNG v18, fix it in two places:

1. Import v18 theme correctly

Remove old v17 imports like:

@import "primeng/resources/themes/lara-light-blue/theme.css";
@import "primeng/resources/primeng.min.css";

Add PrimeNG provider in app.config.ts or app.module.ts:

import { providePrimeNG } from 'primeng/config';
import Aura from '@primeng/themes/aura';
providers: [
  providePrimeNG({
    theme: {
      preset: Aura
    }
  })
]

PrimeNG v18 removed the old primeng/resources theme files, so without the new theme provider, components like p-radioButton display broken.  ￼

Also keep only:

@import "primeicons/primeicons.css";

⸻

2. Fix radio button value type

Your form control value should match boolean, not string.

Use this:

<p-radioButton
  name="contractTemplateAvailable"
  [value]="true"
  formControlName="contractTemplateAvailable"
  inputId="contractTemplateAvailableYes"
  label="Yes"
  adminSelenium="crud_frm_event_contractTemplateAvailable_true">
</p-radioButton>
<p-radioButton
  name="contractTemplateAvailable"
  [value]="false"
  formControlName="contractTemplateAvailable"
  inputId="contractTemplateAvailableNo"
  label="No"
  adminSelenium="crud_frm_event_contractTemplateAvailable_false">
</p-radioButton>

Form initialization:

this.form = this.fb.group({
  contractTemplateAvailable: [null]
});

Or default Yes:

contractTemplateAvailable: [true]

⸻

Important

Do not use:

[value]="'true'"
[value]="'false'"

That makes values string "true" / "false", not boolean.

PrimeNG v18 RadioButton still supports name, value, formControlName, and inputId, but styling depends on the new styled-mode theme setup.  ￼
