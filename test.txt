Yes — you can implement this for your preview API by putting the CSS inside the generated HTML content itself.

But one important clarification:
	•	possible: <style>...</style> inside the HTML string you send to preview API
	•	not possible: true 1.1 / 1.1.1 auto-numbering using only style="" on each tag, because that needs ::before + content: + CSS counters

So the correct approach is:

Use HTML content like this

<style>
  .hierarchical-list,
  .hierarchical-list ol {
    list-style: none;
    margin: 0;
    padding-left: 40px;
    counter-reset: item;
  }

  .hierarchical-list li {
    display: block;
    position: relative;
    margin: 6px 0;
  }

  .hierarchical-list li::before {
    content: counters(item, ".") ". ";
    counter-increment: item;
    position: absolute;
    left: -40px;
    width: 34px;
    text-align: right;
  }
</style>

<ol class="hierarchical-list">
  <li>Hdhfg
    <ol>
      <li>cbvfdjkdf
        <ol>
          <li>Gvhgfdjvh</li>
          <li>vhfvgjdfjv</li>
        </ol>
      </li>
    </ol>
  </li>
  <li>Dfgvjegr</li>
  <li>Higvreugyui</li>
  <li>gjvjdftiue</li>
</ol>

That will render like:
	•	1.
	•	1.1.
	•	1.1.1.
	•	1.1.2.

which matches your screenshot logic.

Step by step implementation

Step 1: Keep TinyMCE nested ordered lists working

Your editor should allow nested <ol> and <li> tags.

Use config like this:

this.editorInit = {
  promotion: false,
  branding: false,
  menubar: 'file edit view insert format tools table help',
  toolbar: 'undo redo | blocks fontfamily fontsize | bold italic underline | alignleft aligncenter alignright | numlist bullist outdent indent | removeformat',
  plugins: 'lists advlist link image table code help wordcount',
  advlist_number_styles: 'default,lower-alpha,lower-roman,upper-alpha,upper-roman',
  advlist_bullet_styles: 'disc,circle,square',
  valid_elements: '*[*]',
  extended_valid_elements: 'style[type],ol[class|style|start|type],ul[class|style],li[class|style],span[class|style]',
  height: 720
};

This is only for editor structure preservation.

⸻

Step 2: Do not depend on content_style for preview output

content_style works inside TinyMCE editor iframe, but that CSS is usually not included in the HTML content returned by:

editor.getContent()

So for your external preview API, you must build final HTML manually.

⸻

Step 3: Create CSS string that will be injected into HTML

Create a method like this:

private getHierarchicalListStyle(): string {
  return `
    <style>
      .hierarchical-list,
      .hierarchical-list ol {
        list-style: none;
        margin: 0;
        padding-left: 40px;
        counter-reset: item;
      }

      .hierarchical-list li {
        display: block;
        position: relative;
        margin: 6px 0;
      }

      .hierarchical-list li::before {
        content: counters(item, ".") ". ";
        counter-increment: item;
        position: absolute;
        left: -40px;
        width: 34px;
        text-align: right;
      }
    </style>
  `;
}


⸻

Step 4: Add class to ordered lists before sending to preview API

When the user clicks preview, take the editor HTML and process it.

private buildPreviewHtml(editorHtml: string): string {
  const parser = new DOMParser();
  const doc = parser.parseFromString(`<div id="root">${editorHtml}</div>`, 'text/html');
  const root = doc.getElementById('root');

  if (!root) {
    return editorHtml;
  }

  const orderedLists = root.querySelectorAll('ol');

  orderedLists.forEach((ol) => {
    ol.classList.add('hierarchical-list');
    ol.removeAttribute('type');
  });

  return `${this.getHierarchicalListStyle()}${root.innerHTML}`;
}


⸻

Step 5: Send processed HTML to preview API

Example:

previewTemplate(): void {
  const rawHtml = this.editor?.getContent() || '';
  const previewHtml = this.buildPreviewHtml(rawHtml);

  const payload = {
    htmlContent: previewHtml
  };

  this.yourService.generatePreview(payload).subscribe({
    next: (res) => {
      console.log('Preview generated', res);
    },
    error: (err) => {
      console.error('Preview failed', err);
    }
  });
}

Step 6: If your preview API uses full HTML document, send full document

Some preview engines render better if you send complete HTML instead of fragment.

Use this version:

private buildFullPreviewHtml(editorHtml: string): string {
  const parser = new DOMParser();
  const doc = parser.parseFromString(`<div id="root">${editorHtml}</div>`, 'text/html');
  const root = doc.getElementById('root');

  if (!root) {
    return editorHtml;
  }

  const orderedLists = root.querySelectorAll('ol');

  orderedLists.forEach((ol) => {
    ol.classList.add('hierarchical-list');
    ol.removeAttribute('type');
  });

  return `
    <!DOCTYPE html>
    <html>
    <head>
      <meta charset="UTF-8" />
      ${this.getHierarchicalListStyle()}
    </head>
    <body>
      ${root.innerHTML}
    </body>
    </html>
  `;
}

If your API accepts full HTML, this is better.

Step 7: If numbering alignment looks off, adjust these values

Inside CSS, these are the important ones:

padding-left: 40px;
left: -40px;
width: 34px;

You can tune them like this:

tighter layout

padding-left: 32px;
left: -32px;
width: 28px;

wider layout

padding-left: 48px;
left: -48px;
width: 40px;


⸻

Important limitation

If your external preview API:
	•	strips <style> tags
	•	sanitizes ::before
	•	ignores CSS counters

then this approach will not work.

In that case, you need manual numbering injection into the HTML text itself, for example:

<p>1. Hdhfg</p>
<p style="margin-left:40px;">1.1. cbvfdjkdf</p>
<p style="margin-left:80px;">1.1.1. Gvhgfdjvh</p>
<p style="margin-left:80px;">1.1.2. vhfvgjdfjv</p>
<p>2. Dfgvjegr</p>

But that is fallback only. First try the CSS-in-HTML approach.
