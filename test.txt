That confirms it: your PDF renderer supports your custom HTML layout, but the DOCX API does not.

So the issue is not your numbering logic anymore.
The issue is:

* for preview/PDF, you are sending decorated HTML
* for DOCX, the external API is interpreting <ol><li> differently and also rendering your custom marker text
* result: native list bullet/number + your manual number text together

That is why Word shows:

* a dot on left
* then 1., 3.1., 3.1.2. as text
* then content below

Your current decorated HTML is something like:

private prepareOrderedListsForDocx(root: HTMLElement): void {
  const orderedLists = Array.from(root.querySelectorAll('ol'));
  for (const list of orderedLists) {
    const ol = list as HTMLOListElement;
    const start = parseInt(ol.getAttribute('start') || '1', 10);
    ol.setAttribute('start', String(start));
    const listStyle = this.getOrderedListStyle(ol);
    const typeMap: Record<OrderedListStyle, string> = {
      decimal: '1',
      'decimal-leading-zero': '1',
      'lower-alpha': 'a',
      'upper-alpha': 'A',
      'lower-roman': 'i',
      'upper-roman': 'I'
    };
    const type = typeMap[listStyle];
    if (type) {
      ol.setAttribute('type', type);
    }
    // keep native list behavior for DOCX engines
    ol.style.listStyleType = listStyle;
    ol.style.paddingLeft = '2rem';
    ol.style.margin = '0 0 0.75rem';
  }
  const items = Array.from(root.querySelectorAll('li'));
  for (const item of items) {
    const li = item as HTMLLIElement;
    li.style.margin = '0.25rem 0';
    li.style.listStyle = '';
  }
  // make p inside li inline-friendly for doc engines
  const paragraphsInsideLi = Array.from(root.querySelectorAll('li > p'));
  for (const p of paragraphsInsideLi) {
    const el = p as HTMLElement;
    el.style.margin = '0';
    el.style.display = 'inline';
  }
  // IMPORTANT: remove preview-only artifacts if they exist
  const previewArtifacts = Array.from(
    root.querySelectorAll('[data-list-index], [data-list-type]')
  );
  for (const artifact of previewArtifacts) {
    artifact.removeAttribute('data-list-index');
    artifact.removeAttribute('data-list-type');
  }
}

⸻

Why PDF works but DOCX fails

Because PDF renderer is browser-like and understands:

* flex
* span-based layout
* custom visual numbering

DOCX renderer usually prefers:

* semantic lists
* minimal CSS
* native <ol>/<li> structure
