Save behavior

On save:

* Update the actual selected table DOM element inside TinyMCE.
* Set/remove attributes accordingly.
* Preserve existing table structure, rows, columns, styles, classes, width, borders, etc.

I want to implement a custom table configuration popup on double-click of a table inside the editor.

Requirement

When a user double-clicks on a table in TinyMCE:

1. Open a custom popup / dialog.
2. In that popup show a dropdown:
    * Table Type
        * Static
        * Dynamic
3. If the user selects Dynamic, then show another dropdown:
    * Entity Type
        * (configurable list; for now use sample values like customer, invoice, product, employee)
4. When the user clicks Save / Apply:
    * Update the selected table element in the TinyMCE content.
    * Add custom attributes to the <table> tag:
        * table-type="static" or table-type="dynamic"
        * if dynamic is selected, also add table-entity-type="selectedEntityType"


<table table-type="dynamic" table-entity-type="customer">
  ...
</table>



Show Table Type dropdown always.
* Show Entity Type dropdown only when table type = dynamic.
* If table type is changed from dynamic to static, remove the table-entity-type attribute.
