The green triangle is coming from old/custom CSS, not from PrimeNG calendar. Some CSS is probably targeting calendar icon with :before / :after and creating triangle shape.

Add this fix in calendar.component.scss:

:host ::ng-deep {
  .ng-wings-calendar {
    width: 100%;
    display: flex;
  }
  .ng-wings-calendar .p-inputtext {
    flex: 1;
    width: 100%;
    height: 38px;
    border-radius: 4px 0 0 4px;
  }
  .ng-wings-calendar .p-datepicker-trigger {
    width: 38px;
    min-width: 38px;
    height: 38px;
    padding: 0;
    background: $theme;
    border: 1px solid $theme;
    color: #fff;
    border-radius: 0 4px 4px 0;
    display: flex;
    align-items: center;
    justify-content: center;
  }
  .ng-wings-calendar .p-datepicker-trigger::before,
  .ng-wings-calendar .p-datepicker-trigger::after,
  .ng-wings-calendar .p-calendar::before,
  .ng-wings-calendar .p-calendar::after {
    content: none !important;
    display: none !important;
    border: 0 !important;
  }
  .ng-wings-calendar .p-datepicker-trigger .p-button-icon,
  .ng-wings-calendar .p-datepicker-trigger .pi {
    color: #fff;
    font-size: 14px;
    margin: 0;
  }
}

Also change your HTML to use button icon mode:

<p-calendar
  #calendar
  [(ngModel)]="value"
  [disabled]="disabled"
  [selectionMode]="selectionMode"
  [attr.id]="id"
  [showIcon]="true"
  iconDisplay="button"
  [dateFormat]="dateFormat"
  [monthNavigator]="true"
  [yearNavigator]="true"
  [yearRange]="yearRange"
  [showTime]="showTime"
  [showSeconds]="showSeconds"
  [hourFormat]="hourFormat"
  styleClass="ng-wings-calendar"
  (onBlur)="touch()"
></p-calendar>

Then search globally for this kind of CSS and remove/override it:

.p-calendar::before
.p-calendar::after
.p-datepicker-trigger::before
.p-datepicker-trigger::after
border-left
border-right
border-bottom

That triangle is almost certainly from one of those rules.
