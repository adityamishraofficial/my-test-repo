It is not working because TinyMCE is still inserting a normal list marker, and your inline-style approach is not replacing the marker generation at editor time.

From your screenshot, that black square means:
	•	the editor is creating a bullet list
	•	or the default list styling is still active
	•	and the custom inline numbering logic has not been applied to the generated HTML yet

For 1, 1.1, 1.1.1 with inline styles only, TinyMCE will not do this automatically from the toolbar button. You need to:
	1.	force ordered list creation
	2.	remove default list style inline
	3.	inject inline <span> numbers into each <li>

Use this exact approach.

1) Use only ordered list button

Make sure you are clicking numbered list, not bullet list.

2) Add this function in your Angular component

applyInlineNestedNumbering(root: HTMLElement) {
  const processOl = (ol: HTMLOListElement, prefix: number[] = []) => {
    ol.style.listStyleType = 'none';
    ol.style.paddingLeft = '24px';
    ol.style.margin = '0';

    const liChildren = Array.from(ol.children).filter(
      (node) => node.tagName.toLowerCase() === 'li'
    ) as HTMLLIElement[];

    liChildren.forEach((li, index) => {
      const currentNumber = [...prefix, index + 1];

      li.style.position = 'relative';
      li.style.listStyleType = 'none';
      li.style.paddingLeft = '32px';
      li.style.margin = '4px 0';

      const directMarkers = Array.from(li.children).filter(
        (node) =>
          node instanceof HTMLElement &&
          node.tagName.toLowerCase() === 'span' &&
          node.getAttribute('data-list-marker') === 'true'
      ) as HTMLSpanElement[];

      directMarkers.forEach((m) => m.remove());

      const marker = document.createElement('span');
      marker.setAttribute('data-list-marker', 'true');
      marker.textContent = currentNumber.join('.') + ' ';
      marker.setAttribute(
        'style',
        'position:absolute;left:0;top:0;white-space:nowrap;'
      );

      li.insertBefore(marker, li.firstChild);

      const nestedOls = Array.from(li.children).filter(
        (node) => node.tagName.toLowerCase() === 'ol'
      ) as HTMLOListElement[];

      nestedOls.forEach((nestedOl) => processOl(nestedOl, currentNumber));
    });
  };

  const topLevelOls = Array.from(root.querySelectorAll('ol')).filter((ol) => {
    return !ol.parentElement || ol.parentElement.tagName.toLowerCase() !== 'li';
  }) as HTMLOListElement[];

  topLevelOls.forEach((ol) => processOl(ol, []));
}

3) Use this TinyMCE config

this.editorInit = {
  promotion: false,
  branding: false,
  menubar: 'file edit view insert format tools table help',
  plugins: 'lists advlist code',
  toolbar: 'undo redo | blocks fontfamily fontsize | bold italic underline | numlist bullist outdent indent | code',
  advlist_number_styles: 'default',
  setup: (editor: any) => {
    const refreshNumbering = () => {
      const body = editor.getBody();
      if (!body) return;

      const allUl = body.querySelectorAll('ul');
      allUl.forEach((ul: HTMLElement) => {
        ul.style.paddingLeft = '';
        ul.style.margin = '';
      });

      const allOl = body.querySelectorAll('ol');
      allOl.forEach((ol: HTMLElement) => {
        ol.style.listStyleType = 'none';
        ol.style.paddingLeft = '24px';
        ol.style.margin = '0';
      });

      const allLi = body.querySelectorAll('li');
      allLi.forEach((li: HTMLElement) => {
        li.style.listStyleType = 'none';
      });

      this.applyInlineNestedNumbering(body);
    };

    editor.on('init', () => {
      setTimeout(refreshNumbering, 100);
    });

    editor.on('SetContent', () => {
      setTimeout(refreshNumbering, 0);
    });

    editor.on('ExecCommand', () => {
      setTimeout(refreshNumbering, 0);
    });

    editor.on('NodeChange', () => {
      setTimeout(refreshNumbering, 0);
    });

    editor.on('KeyUp', () => {
      setTimeout(refreshNumbering, 0);
    });

    editor.on('Change', () => {
      setTimeout(refreshNumbering, 0);
    });
  }
};

6) Why your current attempt fails

The common reasons are:
	•	you clicked bullet list instead of numbered list
	•	list-style-type: none was not applied on <ol> and <li>
	•	the numbering <span> was not injected after TinyMCE changed the DOM
	•	the handler ran too early, before the list existed
