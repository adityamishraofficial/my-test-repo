Your editor is still loading rendered preview HTML like this:

<div>...</div><p><span>Not adding...</span></p>

That is why Enter does not create a new list index. TinyMCE only continues numbering when cursor is inside real:

<ol><li>...</li></ol>

Fix rule

Use 2 separate HTML values:

editorHtml   = real ol/li HTML only
previewHtml  = rendered div/span numbering HTML only

Do not put previewHtml back into TinyMCE.

⸻

Save from TinyMCE

saveTemplate() {
  const editorHtml = this.editor.getContent();
  const cleanEditorHtml = this.cleanEditorHtmlForSave(editorHtml);
  // save this only
  this.payload.html = cleanEditorHtml;
}
private cleanEditorHtmlForSave(html: string): string {
  const div = document.createElement('div');
  div.innerHTML = html;
  // remove preview/rendered html if accidentally present
  div.querySelectorAll('[data-rendered-preview="true"], [data-rendered-list="true"]').forEach((el) => {
    el.remove();
  });
  return div.innerHTML;
}

⸻

Load back into TinyMCE

this.editor.setContent(savedHtml); // savedHtml must be real <ol><li>

Example saved HTML should be:

<ol>
  <li>fsdfsd
    <ol>
      <li>fdsffds</li>
      <li>fdfsfd
        <ol>
          <li>fdsffdsf</li>
        </ol>
      </li>
    </ol>
  </li>
</ol>

Then pressing Enter after fdsffdsf will create the next index automatically.

⸻

Preview/PDF only

openPreview() {
  const savedHtml = this.editor.getContent();
  const previewHtml = this.buildRenderedPreview(savedHtml);
  this.previewBody = previewHtml;
}

Never do this:

this.editor.setContent(previewHtml); // wrong

Your current issue is happening because rendered preview HTML is being inserted inside TinyMCE, so TinyMCE sees normal div/p/span, not list items.
