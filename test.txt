Use real ol/li in TinyMCE, and for both PDF + DOCX use the same cleaned semantic list HTML with minimal inline styles only.

The key is:

* keep native <ol><li>
* do not inject custom marker spans like 1., 2.1.
* do not use flex, table, table-cell
* only set start, type, margins, and padding
* keep nested <ol> inside <li>

That is the most compatible version for:

* TinyMCE editing
* PDF rendering
* DOCX API rendering

Important limitation

With pure native ol/li, most DOCX APIs support:

* 1, 2, 3
* A, B, C
* a, b, c
* I, II, III
* i, ii, iii

But many DOCX HTML engines do not support true hierarchical numbering like 1.1, 1.1.1 from HTML alone.

So this code will give you the best cross-compatible native list behavior, but if your DOCX API specifically requires 1.1.1 visual numbering, that is usually an API limitation, not an HTML one.

Full working code

1. Ordered list type

type OrderedListStyle =
  | 'decimal'
  | 'decimal-leading-zero'
  | 'lower-alpha'
  | 'upper-alpha'
  | 'lower-roman'
  | 'upper-roman';

⸻

2. Main HTML builder for both PDF and DOCX

Use this same method for both files:

private buildExportHtml(content: string): string {
  const root = document.createElement('div');
  root.innerHTML = content;
  root.style.fontFamily = 'Georgia, serif';
  root.style.fontSize = '16px';
  root.style.lineHeight = '1.6';
  root.style.color = '#1f2a37';
  this.applyElementStyles(root);
  this.normalizeOrderedLists(root);
  return root.innerHTML;
}

3. Normalize ordered lists to real ol/li only

private normalizeOrderedLists(root: HTMLElement): void {
  const orderedLists = Array.from(root.querySelectorAll('ol'));
  for (const list of orderedLists) {
    const ol = list as HTMLOListElement;
    const level = this.getOrderedListLevel(ol);
    const style = this.resolveOrderedListStyleByLevel(ol, level);
    const start = parseInt(ol.getAttribute('start') || '1', 10);
    ol.setAttribute('start', String(Number.isNaN(start) ? 1 : start));
    ol.setAttribute('type', this.mapOrderedListStyleToType(style));
    ol.style.listStyleType = style;
    ol.style.marginTop = '0';
    ol.style.marginBottom = '0.75rem';
    ol.style.paddingLeft = `${2 + level * 1.25}rem`;
    // remove preview-only styles/attrs if they were previously added
    ol.style.removeProperty('display');
    ol.style.removeProperty('table-layout');
    ol.style.removeProperty('width');
    ol.style.removeProperty('align-items');
  }
  const listItems = Array.from(root.querySelectorAll('li'));
  for (const item of listItems) {
    const li = item as HTMLLIElement;
    li.style.margin = '0.2rem 0';
    li.style.padding = '0';
    li.style.removeProperty('list-style');
    li.removeAttribute('data-list-index');
    li.removeAttribute('data-list-type');
    // remove preview wrapper div/span if previously generated
    this.unwrapPreviewArtifacts(li);
    // normalize direct paragraphs/divs inside li
    const directChildren = Array.from(li.childNodes);
    for (const child of directChildren) {
      if (!(child instanceof HTMLElement)) {
        continue;
      }
      const tag = child.tagName.toLowerCase();
      if ((tag === 'p' || tag === 'div') && !child.querySelector('ol, ul')) {
        child.style.margin = '0';
        child.style.display = 'inline';
      }
    }
  }
}

4. Remove preview wrappers if old decorated HTML exists

private unwrapPreviewArtifacts(li: HTMLLIElement): void {
  const children = Array.from(li.children);
  if (children.length !== 1) {
    return;
  }
  const first = children[0] as HTMLElement;
  const looksLikePreviewRow =
    first.tagName.toLowerCase() === 'div' &&
    (
      first.style.display === 'flex' ||
      first.style.display === 'table'
    );
  if (!looksLikePreviewRow) {
    return;
  }
  const rowChildren = Array.from(first.children) as HTMLElement[];
  if (rowChildren.length < 2) {
    return;
  }
  const possibleBody = rowChildren[1];
  const fragment = document.createDocumentFragment();
  while (possibleBody.firstChild) {
    fragment.appendChild(possibleBody.firstChild);
  }
  first.replaceWith(fragment);
}

5. Get list level

private getOrderedListLevel(list: HTMLOListElement): number {
  let level = 0;
  let parent = list.parentElement;
  while (parent) {
    if (parent.tagName.toLowerCase() === 'ol') {
      level++;
    }
    parent = parent.parentElement;
  }
  return level;
}

⸻

6. Resolve list style by level

This gives a stable pattern:

* level 0 → decimal
* level 1 → lower-alpha
* level 2 → lower-roman
* level 3 → decimal
* then repeat

private resolveOrderedListStyleByLevel(
  list: HTMLOListElement,
  level: number
): OrderedListStyle {
  const inlineStyle = list.style.listStyleType?.trim().toLowerCase();
  if (this.isSupportedOrderedListStyle(inlineStyle)) {
    return inlineStyle;
  }
  const type = list.getAttribute('type');
  if (type) {
    switch (type) {
      case '1':
        return 'decimal';
      case 'a':
        return 'lower-alpha';
      case 'A':
        return 'upper-alpha';
      case 'i':
        return 'lower-roman';
      case 'I':
        return 'upper-roman';
    }
  }
  const styles: OrderedListStyle[] = [
    'decimal',
    'lower-alpha',
    'lower-roman',
    'decimal'
  ];
  return styles[level % styles.length];
}

7. Supported styles checker

private isSupportedOrderedListStyle(
  value: string | null | undefined
): value is OrderedListStyle {
  return (
    value === 'decimal' ||
    value === 'decimal-leading-zero' ||
    value === 'lower-alpha' ||
    value === 'upper-alpha' ||
    value === 'lower-roman' ||
    value === 'upper-roman'
  );
}

⸻

8. Map style to HTML type

private mapOrderedListStyleToType(style: OrderedListStyle): string {
  switch (style) {
    case 'lower-alpha':
      return 'a';
    case 'upper-alpha':
      return 'A';
    case 'lower-roman':
      return 'i';
    case 'upper-roman':
      return 'I';
    case 'decimal':
    case 'decimal-leading-zero':
    default:
      return '1';
  }
}

9. Apply normal styles, but do not break list items

private applyElementStyles(root: HTMLElement): void {
  const headings = root.querySelectorAll('h1, h2, h3');
  for (const heading of headings) {
    const element = heading as HTMLElement;
    element.style.color = '#1f2a37';
    element.style.fontFamily = 'Georgia, serif';
    element.style.margin = '0 0 0.75rem';
    element.style.lineHeight = '1.15';
  }
  const paragraphs = root.querySelectorAll('p');
  for (const paragraph of paragraphs) {
    const element = paragraph as HTMLElement;
    if (element.closest('li')) {
      element.style.margin = '0';
      element.style.display = 'inline';
    } else {
      element.style.margin = '0 0 0.75rem';
    }
    element.style.color = '#4c5b6b';
  }
  const divs = root.querySelectorAll('li > div');
  for (const div of divs) {
    const element = div as HTMLElement;
    if (!element.querySelector('ol, ul')) {
      element.style.margin = '0';
      element.style.display = 'inline';
    }
  }
}

How to use it

For PDF

const htmlForPdf = this.buildExportHtml(this.editorHtml);

For DOCX

const htmlForDocx = this.buildExportHtml(this.editorHtml);

Use the same HTML for both.

What this code fixes

It fixes:

* broken custom preview wrappers
* flex/table CSS issues in DOCX
* duplicate custom numbering text
* list text appearing on the next line because of <p> wrappers
* preserving real ol/li for TinyMCE Enter behavior

⸻

What it does not guarantee

It does not guarantee true 1.1, 1.1.1 rendering in DOCX if your external DOCX API does not support multilevel HTML lists properly.

That part depends on the DOCX API.

If your API only understands standard HTML list levels, then this code will still produce:

* level 1: 1, 2, 3
* level 2: a, b, c
* level 3: i, ii, iii

which is the most compatible native HTML approach.

⸻

If you need exact 1.1.1 in DOCX too

Then there are only two real options:

* use a DOCX API/library that supports true multilevel numbering definitions
* or generate a separate export-only HTML version

With the same single HTML and real ol/li, exact hierarchical numbering in DOCX is usually not reliable.
